# scrapy-project-setup

## Purpose

Scaffold a Scrapy project from the probe results, wiring in scrapy-poet and scrapy-zyte-api via the addons system.

## Trigger

Load this skill after `scrapy-schema-infer` has run and one or more `zyte-probe-result-<slug>.json` files exist in the cwd.

## Instructions

### 1. Read inputs

- Find all `zyte-probe-result-*.json` files in the cwd.
- For each file read: `url`, `item_type`.
- Use the `url` from any one file to derive the project name and domain (all files share the same domain).

### 2. Derive names

From the `url`:
- Extract the domain (e.g. `www.scan.co.uk` → `scan.co.uk`)
- Strip subdomains (drop anything before the second-to-last dot segment)
- Strip the TLD (drop the last dot segment)
- Slugify: lowercase, replace hyphens and dots with underscores, keep only alphanumeric and underscores

Examples:
- `www.scan.co.uk` → `scan`
- `www.books-to-scrape.com` → `books_to_scrape`
- `news.ycombinator.com` → `ycombinator`

This gives you:
- **project name** — the slugified result (e.g. `scan`)
- **spider name** — same as project name

### 3. Run `scrapy startproject`

```
uv run scrapy startproject <project_name> .
```

The `.` keeps the project flat in the cwd — no extra directory nesting.

This creates:
```
scrapy.cfg
<project_name>/
  __init__.py
  items.py
  middlewares.py
  pipelines.py
  settings.py
  spiders/
    __init__.py
```

### 4. Update `<project_name>/settings.py`

Add the following at the top of the file, after any existing imports:

```python
import os
```

Replace the existing `ROBOTSTXT_OBEY` line generated by `scrapy startproject`:

```python
# Before (generated default):
ROBOTSTXT_OBEY = True

# After:
ROBOTSTXT_OBEY = False
```

Replace the existing concurrency comment block generated by `scrapy startproject`:

```python
# Before (generated default):
# Configure maximum concurrent requests performed by Scrapy (default: 16)
#CONCURRENT_REQUESTS = 32

# After:
CONCURRENT_REQUESTS = 16
# CONCURRENT_REQUESTS_PER_DOMAIN = 1
# DOWNLOAD_DELAY = 1
```

Add the following at the bottom of the file:

```python
ADDONS = {
    "scrapy_zyte_api.Addon": 500,
    "scrapy_poet.Addon": 300,
}

ZYTE_API_KEY = os.environ.get("ZYTE_API_KEY")
```

Do not remove or modify any other existing settings content.

### 5. Create the pages package

Create `<project_name>/pages/__init__.py`. Leave it empty for now — it will be populated by `scrapy-page-objects`.

### 6. Run `scrapy genspider`

```
uv run scrapy genspider <spider_name> <domain>
```

Where `<domain>` is the full registered domain from the URL (e.g. `scan.co.uk`), not the slugified name.

**Spider name collision:** Scrapy does not allow a spider with the same name as the project. If `<spider_name>` equals `<project_name>`, append `_spider` to the spider name (e.g. project `scan` → spider `scan_spider`).

This creates `<project_name>/spiders/<spider_name>.py` using the default basic template.

### 7. Leave `<project_name>/items.py` as-is

Do not modify `items.py`. Items are provided directly by `zyte_common_items` page base classes — no custom item subclasses are needed.

## Rules

- Always use `uv run scrapy`, never call `scrapy` directly.
- Project name and spider name are derived from the domain, not the URL path.
- Never hardcode `ZYTE_API_KEY` — always read from `os.environ.get("ZYTE_API_KEY")`.
- Use `scrapy_poet.Addon` (not `InjectionMiddleware`) as the ADDONS key.
- The `pages/` directory must be created as a Python package (with `__init__.py`).
- Do not create custom item subclasses in `items.py` — leave it untouched.
- If the spider name would equal the project name, append `_spider` to avoid the Scrapy collision error.
- The `.` argument to `scrapy startproject` is required — do not omit it.

## Output

```
scrapy.cfg
<project_name>/
  __init__.py
  items.py                  ← untouched (not used)
  middlewares.py
  pipelines.py
  settings.py               ← ROBOTSTXT_OBEY=False, CONCURRENT_REQUESTS=16, ADDONS + ZYTE_API_KEY added
  pages/
    __init__.py             ← empty, populated later by scrapy-page-objects
  spiders/
    __init__.py
    <spider_name>.py        ← generated by scrapy genspider
```
